课程总览CUDA Foundation Course

Lesson 03 · 性能诊断语言

CUDA 性能模型导论

第三节是诊断课。第二节已经让 vector add 能跑起来，这一节要回答：结果正确以后，怎样判断慢在哪里，怎样把 H2D/kernel/D2H、read/compute/write、byte/FLOP/bandwidth 放到同一套语言里。

这一节为什么这样简化

本节刻意不进入 float2、float4、half2 的代码实现。它只把判断链路讲稳：先拆总时间，再放大 kernel 内部，再算 AI，再用 Roofline 和 NCU 给出优化方向。具体动作留给第四节。

PPT 故事线

  1. 从能跑到跑得好：先说明为什么性能是成本、等待和体验问题。
  2. 从 CUDA 总耗时进入 H2D、kernel、D2H，再把 Device 内部放大为 read/compute/write。
  3. 建立 byte、FLOP、FLOPS、bandwidth、AI 的共同语言。
  4. 把 vector add 放进 Roofline，判断更像 memory-bound。
  5. 最后说明 NCU 如何把理论判断落到单 kernel 证据。
